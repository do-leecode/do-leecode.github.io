    unikernel基本概念

    • 应用与内核处于同一特权级
    • 共享同一地址空间
    • 编译形成一个Image一体运行

    unikernel组件化开发演变

    • 直接开发一个裸机应用满足输出hello简单需求
    • 需求增加，发现可按照通用性和Arch相关性实现软件代码的分层复用
    • 引入组件化，降低耦合性，提升复用性、定制性、灵活性

    课后练习1：支持带颜色的打印输出

    实现思路：修改axlog的lib.rs中的print_fmt方法如下：

    pub fn print_fmt(args: fmt::Arguments) -> fmt::Result {
    use kspin::SpinNoIrq; // TODO: more efficient
    static LOCK: SpinNoIrq<()> = SpinNoIrq::new(());
    let _guard = LOCK.lock();
    Logger.write_fmt(with_color!(ColorCode::Green,"{}",args))
}

    修改ulib/axstd/src/io/stdio.rs中的__print_impl方法：

    macro_rules! with_color {
    ($color_code:expr, $($arg:tt)*) => {{
        format_args!("\u{1B}[{}m{}\u{1B}[m", $color_code as u8, format_args!($($arg)*))
    }};
}

#[doc(hidden)]
pub fn __print_impl(args: core::fmt::Arguments) {
    if cfg!(feature = "smp") {
        // synchronize using the lock in axlog, to avoid interleaving
        // with kernel logs
        arceos_api::stdio::ax_console_write_fmt(with_color!(34,"{}",args)).unwrap();
    } else {
        stdout().lock().write_fmt(with_color!(34,"{}",args)).unwrap();
    }
}

    这里我设置打印颜色为蓝色，所以设置为34。
    颜色枚举如下：

    enum ColorCode {
